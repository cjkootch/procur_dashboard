# Commercial Strategy — VTC's Niche and Operating Model

**Canonical document:** [`cjkootch/vex` → `docs/commercial-strategy.md`](https://github.com/cjkootch/vex/blob/main/docs/commercial-strategy.md)

The canonical commercial strategy document lives in vex because the
commercial frame primarily governs how vex's execution layer engages
counterparties. Procur is the intelligence warehouse that supports
the commercial strategy; it does not define it.

## What this document captures

- VTC's commercial niche: sub-major Caribbean / LatAm physical
  commodity trading at $1-5M cargo size
- Three sub-niches: Caribbean fuel (cash engine), specialty crude
  single-cargo (high-margin track), sub-floor referrals from majors
  (deferred)
- "Selling without selling" — the system as observable behavior,
  not as product to demo
- Outreach strategy: shotgun coverage with rifle precision per
  interaction, 120-180 active relationships, sustained operational
  rhythm
- Discipline rules that govern commercial execution

## Why this matters for procur

Procur's role is to support the commercial strategy by providing:
- Counterparty intelligence (the data behind "specificity")
- Pricing analytics (the data behind "empirical price quoting")
- Cargo + entity intelligence (the data behind "we know things they
  didn't tell us")
- Match queue (the system behind sustained-volume specific outreach)

When prioritizing procur features, the test is: does this serve the
commercial strategy described in the canonical document? If yes,
ship. If no, defer.
